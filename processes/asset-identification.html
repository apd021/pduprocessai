Asset Identification

Clause: 15.3

Objective

Identify assets within the item that require protection based on their cybersecurity properties and value.

Inputs

  • Item definition
  • Architecture description
  • Data flow diagrams

Outputs / Work Products

  • WP-15-01: Asset identification report
  • WP-15-02: Asset catalog
  • WP-15-03: Damage scenarios

Requirements

  • [RQ-15-01] Assets shall be identified based on cybersecurity properties
  • [RQ-15-02] Damage scenarios shall be defined for each asset
  • [RQ-15-03] Assets shall be categorized and documented

Key Activities

  • Identify functions and data requiring protection
  • Analyze cybersecurity properties (CIA) for each asset
  • Define damage scenarios for asset compromise
  • Map assets to system components
  • Categorize assets by criticality
  • Document asset dependencies

Asset Categories

  • Data assets: Sensitive data, cryptographic keys, credentials
  • Function assets: Safety-critical functions, security controls
  • Component assets: ECUs, sensors, actuators
  • Communication assets: Interfaces, protocols, channels

Cybersecurity Properties

  • Confidentiality: Protection from unauthorized disclosure
  • Integrity: Protection from unauthorized modification
  • Availability: Protection from denial of service
