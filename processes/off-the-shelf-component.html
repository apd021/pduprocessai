Off-the-Shelf Component

Clause: 6.4.6

Objective

Evaluate and manage cybersecurity aspects of components not developed according to ISO/SAE 21434.

Inputs

  • Component specifications
  • Vendor documentation
  • Vulnerability databases
  • Project cybersecurity requirements

Outputs / Work Products

  • WP-06-13: Off-the-shelf component analysis
  • WP-06-14: Component vulnerability assessment
  • WP-06-15: Integration requirements

Requirements

  • [RQ-06-21] Off-the-shelf components shall be evaluated for cybersecurity
  • [RQ-06-22] Known vulnerabilities shall be analyzed
  • [RQ-06-23] Suitability for the intended use shall be determined
  • [RQ-06-24] Mitigations for identified weaknesses shall be implemented

Key Activities

  • Collect available cybersecurity information from the vendor
  • Search vulnerability databases (CVE, NVD) for known issues
  • Assess component against project cybersecurity requirements
  • Identify gaps and required mitigations
  • Define integration and monitoring requirements
  • Establish process for vulnerability updates from vendor
