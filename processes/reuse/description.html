Reuse

Clause: 6.4.4

Objective

Ensure that previously developed components are suitable for reuse in new projects from a cybersecurity perspective.

Inputs

  • Component documentation from previous project
  • Cybersecurity case of original component
  • New project requirements

Outputs / Work Products

  • WP-06-07: Reuse analysis report
  • WP-06-08: Gap analysis
  • WP-06-09: Adapted cybersecurity case

Requirements

  • [RQ-06-13] Reused components shall be analyzed for cybersecurity suitability
  • [RQ-06-14] Differences between original and new application shall be identified
  • [RQ-06-15] Additional cybersecurity activities shall be performed if needed
  • [RQ-06-16] Cybersecurity documentation shall be updated for reuse

Key Activities

  • Analyze the original cybersecurity case and documentation
  • Compare original and new operational environments
  • Identify gaps in cybersecurity coverage
  • Perform delta threat analysis and risk assessment
  • Implement necessary modifications or additional controls
  • Update cybersecurity case for the new application
