Attack Path Analysis

Clause: 15.6

Objective

Analyze potential attack paths that could be used to realize threat scenarios, from entry point to target asset.

Inputs

  • Threat scenarios
  • System architecture
  • Component interfaces

Outputs / Work Products

  • WP-15-08: Attack path analysis
  • WP-15-09: Attack trees / Attack graphs

Requirements

  • [RQ-15-10] Attack paths shall be analyzed for threat scenarios
  • [RQ-15-11] Entry points and attack vectors shall be identified
  • [RQ-15-12] Attack steps shall be documented

Key Activities

  • Identify system entry points
  • Analyze interfaces and attack vectors
  • Develop attack trees or graphs
  • Identify intermediate steps and lateral movement
  • Consider existing security controls
  • Document complete attack paths

Common Attack Vectors

  • Physical: OBD-II, debug ports, USB
  • Wireless: Bluetooth, WiFi, cellular, V2X
  • In-vehicle networks: CAN, LIN, Ethernet
  • Backend: Cloud services, mobile apps
  • Supply chain: Compromised components, updates

Attack Path Components

  • Entry point (initial access)
  • Attack steps (exploitation, lateral movement)
  • Target asset (final objective)
  • Required capabilities and resources
  • Detection opportunities
