Engineering Translation: Threat Scenario Identification

Technical Implementation

Systematically identify potential threat scenarios using structured threat modeling methodologies and automotive-specific threat intelligence.

Implementation Approach

  • Threat Modeling: Apply STRIDE methodology (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege)
  • Attack Tree Analysis: Build attack trees showing paths from entry points to assets
  • Threat Intelligence: Leverage Auto-ISAC, CVE databases, and security research
  • Automotive Context: Consider vehicle-specific threats (remote access, CAN injection, key fob attacks)
  • Threat Actor Profiles: Define attacker capabilities, motivations, and resources

Technical Specifications

  • Threat scenario catalog template with fields: ID, description, threat actor, attack vector, target asset
  • STRIDE classification matrix
  • Attack tree diagrams
  • Threat actor capability matrix
  • Threat-to-damage mapping

Engineering Artifacts

  • WP-15-04: Threat scenario catalog
  • WP-15-05: Threat actor analysis
  • Attack tree diagrams
  • Threat intelligence reports

Tools and Methods

  • Threat modeling tools (Microsoft TMT, OWASP Threat Dragon)
  • Attack tree tools (SecuriTree, AttackTree)
  • Threat intelligence platforms (Auto-ISAC, CVE databases)
  • Diagramming tools

Threat Actor Categories

  • Script Kiddies: Low skill, opportunistic, use existing tools
  • Hacktivists: Ideologically motivated, medium capability
  • Criminals: Financially motivated, medium-high capability
  • Competitors: Business advantage, high capability
  • Nation States: Strategic goals, very high capability
  • Insiders: Authorized access, various motivations
