Cybersecurity Event Evaluation

Clause: 7.4.2

Objective

Evaluate detected cybersecurity events to determine their relevance and impact on the organization's products.

Inputs

  • Detected cybersecurity events
  • Product information
  • Previous vulnerability assessments

Outputs / Work Products

  • WP-07-04: Event evaluation report
  • WP-07-05: Weakness identification
  • WP-07-06: Triage decision records

Requirements

  • [RQ-07-05] Detected events shall be evaluated for relevance
  • [RQ-07-06] Weaknesses in products shall be identified
  • [RQ-07-07] Triage decisions shall be documented
  • [RQ-07-08] Relevant events shall be forwarded for analysis

Key Activities

  • Assess relevance of detected events to products
  • Correlate events with product components (SBOM)
  • Perform initial impact assessment
  • Categorize and prioritize events
  • Determine if vulnerability analysis is needed
  • Forward relevant events for detailed analysis

Triage Categories

  • Not applicable: Event does not affect products
  • Analysis required: Detailed vulnerability analysis needed
  • Immediate action: Critical event requiring urgent response
