Attack Feasibility Rating

Clause: 15.7

Objective

Assess the feasibility of attack paths based on attacker capabilities, resources, and existing security controls.

Inputs

  • Attack path analysis
  • Threat actor profiles
  • Security controls

Outputs / Work Products

  • WP-15-10: Attack feasibility rating
  • WP-15-11: Feasibility analysis documentation

Requirements

  • [RQ-15-13] Attack feasibility shall be rated
  • [RQ-15-14] Rating method shall be defined and applied consistently
  • [RQ-15-15] Feasibility ratings shall be justified

Key Activities

  • Select feasibility rating method
  • Assess attack parameters for each path
  • Consider attacker expertise and resources
  • Evaluate effectiveness of security controls
  • Determine overall feasibility rating
  • Document assessment rationale

Attack Potential Method (ISO/IEC 18045)

Factor Description
Elapsed time Time required to execute attack
Expertise Technical skill level needed
Knowledge Information about target required
Window of opportunity Access conditions needed
Equipment Tools and resources required

Feasibility Levels

  • High: Attack is easily achievable
  • Medium: Attack requires moderate effort
  • Low: Attack requires significant effort
  • Very Low: Attack is extremely difficult
