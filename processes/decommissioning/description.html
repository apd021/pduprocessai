Decommissioning

Clause: 14.4.2

Objective

Ensure secure decommissioning of products to protect sensitive data and prevent misuse of security assets.

Inputs

  • End of cybersecurity support plan
  • Security asset inventory
  • Data protection requirements

Outputs / Work Products

  • WP-14-04: Decommissioning procedures
  • WP-14-05: Data disposal records
  • WP-14-06: Decommissioning verification

Requirements

  • [RQ-14-05] Decommissioning procedures shall be defined
  • [RQ-14-06] Sensitive data shall be securely disposed
  • [RQ-14-07] Security assets shall be invalidated
  • [RQ-14-08] Decommissioning shall be verified

Key Activities

  • Define decommissioning procedures
  • Identify data requiring secure deletion
  • Revoke cryptographic keys and certificates
  • Disable security-sensitive functions
  • Erase personal and sensitive data
  • Verify complete decommissioning

Decommissioning Checklist

  • Personal data erased (GDPR compliance)
  • Cryptographic keys zeroized
  • Certificates revoked
  • Backend connections deregistered
  • Debug interfaces secured
  • Documentation updated
