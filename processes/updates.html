Updates

Clause: 13.4.2

Objective

Manage cybersecurity-related updates to products in the field to address vulnerabilities and maintain security.

Inputs

  • Vulnerability management decisions
  • Remediation requirements
  • Update packages

Outputs / Work Products

  • WP-13-04: Update management procedures
  • WP-13-05: Update deployment records
  • WP-13-06: Update verification results

Requirements

  • [RQ-13-05] Update mechanisms shall be established
  • [RQ-13-06] Updates shall be securely distributed
  • [RQ-13-07] Update installation shall be verified
  • [RQ-13-08] Rollback capability shall be provided

Key Activities

  • Define update distribution mechanisms (OTA, service)
  • Implement secure update delivery
  • Validate updates before deployment
  • Monitor update deployment status
  • Verify successful installation
  • Maintain update records

Update Security Requirements

  • Authenticated update source
  • Integrity verification of update package
  • Confidentiality of update contents (if applicable)
  • Secure installation process
  • Rollback protection and capability
  • Version and compatibility checking
