Engineering Translation: Cybersecurity Goals

Technical Implementation

Derive specific, measurable cybersecurity goals from TARA results and allocate them to system components with assigned CAL levels.

Implementation Approach

  • Goal Derivation: Translate risk treatment decisions into concrete security goals
  • CAL Assignment: Assign Cybersecurity Assurance Level (1-4) based on risk value
  • Goal Allocation: Map goals to specific components and functions
  • Verification Criteria: Define testable criteria for each goal
  • Traceability: Maintain links from goals to threat scenarios and requirements

Technical Specifications

  • Cybersecurity goal template: ID, description, CAL, allocated component, verification method
  • CAL assignment matrix (risk value → CAL mapping)
  • Goal-to-component allocation matrix
  • Verification criteria for each goal
  • Traceability matrix

Engineering Artifacts

  • WP-09-04: Cybersecurity goals specification
  • WP-09-05: Goal allocation to components
  • CAL assignment rationale
  • Verification test specifications

Cybersecurity Assurance Levels (CAL)

  • CAL 1: Low assurance - Basic security measures, standard development practices
  • CAL 2: Moderate assurance - Enhanced security measures, formal reviews
  • CAL 3: High assurance - Rigorous security measures, independent assessment
  • CAL 4: Highest assurance - Comprehensive security measures, formal verification

Tools and Methods

  • Requirements management tools (DOORS, Jira)
  • Traceability tools
  • Goal specification templates
  • CAL assignment guidelines
