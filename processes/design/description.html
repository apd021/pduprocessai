Design

Clause: 10.4.1

Objective

Implement cybersecurity requirements in the design of hardware and software components.

Inputs

  • Cybersecurity concept
  • Cybersecurity requirements
  • Architecture specifications

Outputs / Work Products

  • WP-10-01: Cybersecurity design specification
  • WP-10-02: Design verification results
  • WP-10-03: Design review records

Requirements

  • [RQ-10-01] Cybersecurity requirements shall be implemented in design
  • [RQ-10-02] Secure design principles shall be applied
  • [RQ-10-03] Design shall be verified against requirements
  • [RQ-10-04] Design reviews shall be conducted

Key Activities

  • Implement security controls in HW/SW design
  • Apply secure coding standards (CERT, MISRA)
  • Conduct design reviews for security
  • Perform static analysis and code review
  • Implement cryptographic functions correctly
  • Document security design decisions

Design Principles

  • Defense in depth
  • Least privilege
  • Secure defaults
  • Fail-safe design
  • Separation of duties
  • Minimizing attack surface
