Component Out-of-Context

Clause: 6.4.5

Objective

Develop components independently of a specific item, with defined assumptions that can be verified during integration.

Inputs

  • Component requirements
  • Assumed cybersecurity requirements
  • Market analysis and use case assumptions

Outputs / Work Products

  • WP-06-10: Component cybersecurity requirements
  • WP-06-11: Assumed cybersecurity context
  • WP-06-12: Component cybersecurity case

Requirements

  • [RQ-06-17] Assumptions for out-of-context development shall be defined
  • [RQ-06-18] Assumed cybersecurity requirements shall be documented
  • [RQ-06-19] Assumptions shall be verifiable during integration
  • [RQ-06-20] Component cybersecurity case shall document all assumptions

Key Activities

  • Define assumed operational environment
  • Establish assumed cybersecurity requirements and interfaces
  • Document assumed threat scenarios and attack paths
  • Develop component with assumed security context
  • Create component cybersecurity case
  • Define verification criteria for assumption validation
