Information Security Management

Clause: 5.4.6

Objective

Manage information security to protect work products and cybersecurity-related information from unauthorized access and modification.

Inputs

  • Information classification requirements
  • Access control requirements
  • Work product protection requirements

Outputs / Work Products

  • WP-05-13: Information security management documentation
  • WP-05-14: Access control records
  • WP-05-15: Information classification scheme

Requirements

  • [RQ-05-18] Information security shall be managed for work products
  • [RQ-05-19] Information shall be classified based on sensitivity
  • [RQ-05-20] Access to sensitive information shall be controlled
  • [RQ-05-21] Work products shall be protected according to their classification

Key Activities

  • Define information classification levels (e.g., confidential, restricted, public)
  • Implement access control mechanisms
  • Protect source code and cryptographic materials
  • Secure storage and transmission of sensitive data
  • Manage cryptographic keys and certificates
  • Conduct information security risk assessments
